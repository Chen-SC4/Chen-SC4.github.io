Hexo图片问题 处理方式1

Hexo图片问题 处理方式1

初次搭建博客，暂时使用GitHub来做图床，计划后续迁移到七牛云。

下载PicGo

下载PicGo用来进行图片上传

在GitHub中新建一个仓库

在GitHub中新建一个仓库，用来存放静态资源。仓库的名字可以任意。

配置PicGo

  • 设定仓库名 填写用户名/仓库名
  • 分支名 默认填写为main
  • Token用于验证个人身份
    • 点击GitHub右上角个人头像 settings
    • 点击左侧菜单最底部的Developer Settings
      • 点击左侧菜单中的 Personal access tokens
      • 点击 Generate new token
      • Note 中填写token的使用目的
      • Expiration表示token的有效时间 可以选择永久
      • Select Scope选择第一个repo即可
  • 设定自定义域名 该项为可选项
    • 如果不填 从国内访问博客 图片加载可能会很慢
    • 可以填写一个免费的CDN加速 https://cdn.jsdelivr.net/gh/GitHub用户名/仓库名
    • jsdelivr有下线的风险，推荐后续将图片资源转移到自己的服务器上

点击确定 配置完成

之后只要截取图片 使用PicGo上传，然后选择链接格式，再在相册中找到对应的图片，点击复制链接即可。
